redis5-基础数据结构：跳跃表

发表于 2021-03-07 | 分类于 redis

跳跃表，是redis有序集合的底层实现。主要是为了兼顾插入删除和查询之间的性能。相比数组，数组不利于插入和删除；相比链表，链表的查询效率低下；相比平衡树和红黑树，这两个实现复杂度极高。跳跃表效率堪比红黑树，但实现却远比其简单。

数据结构

跳跃表可以看做分层的有序链表。

如上图，假设现在需要查询30。
1）从第三层开始，1节点比30小，向后比较。
2）55节点比30大，所以从1节点下降一层继续向后比较。
3）在第二层，第20节点比30小，继续向后比较。
4）55节点比30大，所以从20节点下降一层继续向后比较。
5）在第一层，找到第30节点。

综上可以看出，可以通过将节点分层，查询时第最高层向后查找，直到节点的值为NULL或者比目标值大时，回到上一个节点，下降一层，继续向后遍历，直到找到目标节点。在节点数量很多时候，能跳过很多节点，查询效率得到大大的提升，这就是跳跃表的实现基本思想。本质上，就是以牺牲空间来换取查询的速度。

redis中跳跃表的实现：

1. 跳跃表由很多层组成，每层是一个有序的链表。
2. 跳跃表有一个header头节点，节点中有一个64层的结构，每层的结构包含指向一下节点的指针，当前节点到下个节点所跨越的节点个数为本层的跨度(span)。
3. 除了头节点，层数最高的节点的层高称为跳跃表的高度(level)。
4. 除了头节点，一个元素在上层有序链表出现过，那么它一定在下层的有序链表中出现。
5. 每层的最后一个节点指向NULL。
6. 跳跃表有一个尾节点指针，指向最后一个节点。
7. 第一层的节点数量称跳跃表的长度(不包括头节点)。
8. 每一个节点包含一个后指针，指向前一节点，header和第一个节点指向NULL。

跳跃表节点结构体：

跳跃表中所有分值都是按照从小到大排序的，当分值相同时，节点会按照key值(member)排序。

跳跃表结构体：

跳跃表中，可以在O(1)时间复杂度内，快速获取到跳跃表的头尾节点，长度和高度。

跳跃表的查询、插入、删除的平均复杂度都为O(logN)
