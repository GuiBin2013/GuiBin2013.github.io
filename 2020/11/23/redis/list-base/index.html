底层数据结构

List对象的底层实现是quicklist，即快速列表，是linkedlist双端链表以及ziplist压缩列表的组合。

List每个节点的对象结构体

双端链表linkedlist

特点：

1.节点带有前后节点，获取前后置节点时间复杂度为O(1)
2.带有链表头尾指针，len属性的存在使得获取链表长度为O(1)

压缩列表ziplist

ziplist是为了节省内存而出现的，一系列特殊编码的连续内存块，组成的顺序型数据结构。与双端链表相比，ziplist具有节省内存空间，但增加了删除和更新时的复杂度，在节点数量较小时才比较合适。

快速列表quicklist

由于双向链表每个节点需要额外的的pre和tail的空间(16个字节)，并且每个节点的内存独立分配，不利于管理。quicklist采用按段划分，每段之间采用linkedlist关联，而段内采用ziplist进行管理。

总结

  • 双端： 链表节点带有 prev 和 next 指针， 获取某个节点的前置节点和后置节点的复杂度都是 O(1) 。
  • 无环： 表头节点的 prev 指针和表尾节点的 next 指针都指向 NULL ， 对链表的访问以 NULL 为终点。
  • 带表头指针和表尾指针： 通过 list 结构的 head 指针和 tail 指针， 程序获取链表的表头节点和表尾节点的复杂度为 O(1) 。
  • 带链表长度计数器： 程序使用 list 结构的 len 属性来对 list 持有的链表节点进行计数， 程序获取链表中节点数量的复杂度为 O(1) 。
  • 多态： 链表节点使用 void* 指针来保存节点值， 并且可以通过 list 结构的 dup 、 free 、 match 三个属性为节点值设置类型特定函数， 所以链表可以用于保存各种不同类型的值
